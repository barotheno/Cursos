CONMUTADORES NMAP:

   -sS          =   Syn Scan
   -sU          =   Escaneo UDP
   -O           =   Sistema operativo informacion
   -sV          =   Detectar servicios
   -oA          =   Exportar información a 3 archivos diferentes 
   -oA          =   Formato no grepeable
   -oG          =   Formato grepeable 
   -A           =   Modo agresivo
   -T5          =   Velocidad
   -p 50        =   Puerto especifico
   -p-          =   todos los puertos
   --script     =   Seleccionar un script 
   --script=vuln=   Por categorias


TIPOS DE ESCANEOS

    Cuando se escanea un puerto con Nmap, hay tres tipos básicos de escaneo. Estos son:

            Exploraciones de TCP Connect ( -sT)
            Escaneos SYN "semiabiertos" ( -sS)
            Escaneos UDP ( -sU)

Además, hay varios tipos de escaneo de puertos menos comunes, algunos de los cuales también cubriremos (aunque con menos detalle). Estos son:

            Escaneos nulos de TCP ( -sN)
            Exploraciones TCP FIN ( -sF)
            Escaneos de Navidad de TCP ( -sX)

SCRIPTS

    CATEGORIAS 

        safe: - No afectará al objetivo
        intrusive: - No seguro: es probable que afecte al objetivo
        vuln: - Escanear en busca de vulnerabilidades
        exploit: - Intento de aprovechar una vulnerabilidad
        auth: - Intente omitir la autenticación para los servicios en ejecución (por ejemplo, inicie sesión en un servidor FTP de forma anónima)
        brute: - Intento de usar la fuerza bruta de las credenciales para ejecutar servicios
        discovery: - Intente consultar los servicios en ejecución para obtener más información sobre la red (por ejemplo, consultar un servidor SNMP)

    TRABAJANDO CON EL NSE

        Para introducir argumentos, en los escript que los requieran:
            nmap -p 80 --script http-put --script-args http-put.url='/dav/shell.php',http-put.file='./shell.php'
        Para tener información sobre un script:
            nmap --script-help <script-name>

EVASION DE CORTAFUEGOS

    -Pn         =   Evita hacer ping antes de el escaneo 
    -f: - Se utiliza para fragmentar los paquetes (es decir, dividirlos en partes más pequeñas) haciendo menos probable que los paquetes sean 
            detectados por un firewall o IDS.
        Una alternativa a -f, pero que proporciona más control sobre el tamaño de los paquetes: --mtu <number>acepta un tamaño máximo de unidad 
        de transmisión para usar en los paquetes enviados. Este debe ser un múltiplo de 8.
        --scan-delay <time>ms: - utilizado para agregar un retraso entre los paquetes enviados. Esto es muy útil si la red es inestable, pero también 
        para evadir cualquier firewall basado en el tiempo / disparador de IDS que pueda estar en su lugar.
        --badsum: - se utiliza para generar una suma de comprobación no válida para paquetes. Cualquier pila de TCP / IP real eliminaría este paquete; 
        sin embargo, los firewalls pueden responder automáticamente sin molestarse en verificar la suma de verificación del paquete. Como tal, este conmutador 
        se puede utilizar para determinar la presencia de un firewall / IDS.
