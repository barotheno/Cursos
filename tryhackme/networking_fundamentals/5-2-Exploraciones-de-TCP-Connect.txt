Exploraciones de TCP Connect

Para comprender los escaneos de TCP Connect ( -sT), es importante que se sienta cómodo con el protocolo de enlace de tres vías de TCP . Si este término es nuevo 
para usted, sería recomendable completar Introductory Networking antes de continuar.

Como breve recapitulación, el apretón de manos de tres vías consta de tres etapas. Primero, el terminal de conexión (nuestra máquina atacante, en este caso) 
envía una solicitud TCP al servidor de destino con la bandera SYN activada. El servidor luego reconoce este paquete con una respuesta TCP que contiene el 
indicador SYN, así como el indicador ACK. Finalmente, nuestra terminal completa el protocolo de enlace enviando una solicitud TCP con la bandera ACK activada.



Este es uno de los principios fundamentales de las redes TCP / IP, pero ¿cómo se relaciona con Nmap?

Bueno, como sugiere el nombre, un escaneo de TCP Connect funciona realizando el protocolo de enlace de tres vías con cada puerto de destino por turno. En 
otras palabras, Nmap intenta conectarse a cada puerto TCP especificado y determina si el servicio está abierto por la respuesta que recibe.

Por ejemplo, si un puerto está cerrado, RFC 793 establece que:

"... Si la conexión no existe (CERRADA), se envía un reinicio en respuesta a cualquier segmento entrante excepto a otro reinicio. En particular, los SYN 
dirigidos a una conexión inexistente son rechazados por este medio".

En otras palabras, si Nmap envía una solicitud TCP con el indicador SYN configurado en un puerto cerrado , el servidor de destino responderá con un paquete 
TCP con el indicador RST (Reset) configurado. Con esta respuesta, Nmap puede establecer que el puerto está cerrado.



Sin embargo, si la solicitud se envía a un puerto abierto , el objetivo responderá con un paquete TCP con los indicadores SYN / ACK configurados. Luego, 
Nmap marca este puerto como abierto (y completa el protocolo de enlace enviando un paquete TCP con ACK configurado).

Todo esto está muy bien, sin embargo, existe una tercera posibilidad.

¿Qué pasa si el puerto está abierto, pero oculto detrás de un firewall?

Muchos cortafuegos están configurados para eliminar simplemente los paquetes entrantes. Nmap envía una solicitud TCP SYN y no recibe nada a cambio. 
Esto indica que el puerto está protegido por un firewall y, por lo tanto, se considera que el puerto está filtrado .

Dicho esto, es muy fácil configurar un firewall para responder con un paquete RST TCP. Por ejemplo, en IPtables para Linux, una versión simple del 
comando sería la siguiente:

iptables -I INPUT -p tcp --dport <port> -j REJECT --reject-with tcp-reset

Esto puede hacer que sea extremadamente difícil (si no imposible) obtener una lectura precisa de los objetivos.
