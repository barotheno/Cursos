&& == Significa "y" es decir hacerto y despues de que el primero se alla ejecutado con exito.
& == Ejecuta comandos en segundo plano.
$ == El $ es un operador inusualmente especial, ya que se usa para denotar variables de entorno. 
        Estas son variables establecidas por la computadora (puede configurarlas usted mismo, pero entraremos en eso) 
        que se utilizan para afectar diferentes procesos y cómo funcionan. Lo que significa que si edita estas variables, 
        puede cambiar la forma en que ciertos procesos funcionan en su computadora. Por ejemplo, su usuario actual siempre 
        se almacena en una variable de entorno llamada $ USER. Puede ver estas variables con el comando echo.

        Naturalmente, esto significa que las variables de entorno también se pueden usar como entrada para otros comandos, por ejemplo, 
        digamos que quería crear un archivo que sea el nombre de nuestro usuario actual, podría hacerlo touch $USER.



        Recuerde que el operador >> agrega salida a un archivo.

        Las variables de entorno también se pueden configurar con bastante facilidad, solo ejecutarlas export <varname>=<value>establecerá 
        eso como una variable de entorno

| ==    Continuando con la tendencia de operadores muy especiales, tenemos la tubería. La tubería es única porque mientras operadores como >> le permiten almacenar 
        la salida de un comando, el | El operador le permite tomar la salida de un comando y usarlo como entrada para un segundo comando.

        Por ejemplo, puedo usar catpara obtener la salida de un archivo y canalizarlo greppara buscar una cadena específica 
        (Nota: aprenderemos más sobre grep más adelante, pero por ahora solo sepa que es un comando que se usa para encontrar cadenas específicas en una entrada).  

 

        Vale la pena señalar que no todos los comandos admiten la tubería, y algunos que la admiten requieren que los use, en lugar de la entrada, por ejemplo cat -. 
        Así que siempre verifique si el comando lo admite. 

; ==    El ; El operador funciona de manera muy similar a &&, sin embargo, no requiere que el primer comando se ejecute correctamente. 
        Esto significa que puede hacerlo dkhsgffgsafgfasdgfasfghkgdsgfs; lsy aún verá la salida de ls.



        Lea lo anterior.

> ==    > es el operador para la redirección de salida. Lo que significa que puede redirigir la salida de cualquier comando a un archivo. Por ejemplo, 
        si ejecutara echo hello> file, en lugar de enviar hello a la consola, guardaría ese resultado en un archivo llamado file.



        Vale la pena señalar que si usara este operador en un archivo que ya existe, borraría completamente el contenido de ese archivo y lo 
        reemplazaría con la salida de su comando

>> ==   >> hace principalmente lo mismo que>, con una diferencia clave. >> agrega la salida de un comando a un archivo, en lugar de borrarlo.
