A lo largo de esta sala, ha visto muchos archivos y directorios, así que estoy usando esta tarea para definir qué hacen los más importantes de esos archivos y 
directorios.

Sin embargo, antes de eso, vale la pena señalar exactamente cómo funciona el sistema de archivos de Linux. Todo en el sistema de archivos de 
Linux se extiende desde "/". / es el equivalente de C: en Windows. Lo que significa que, por ejemplo, si eliminara "/", eliminaría todos los archivos de su sistema.

/ etc / passwd: almacena la información del usuario: a menudo se usa para ver a todos los usuarios en un sistema

    

/ etc / shadow: tiene todas las contraseñas de estos usuarios

    No se muestra por razones obvias

/ tmp: todos los archivos que contiene se eliminan cuando se apaga; se usa para archivos temporales

    

/ etc / sudoers: se utiliza para controlar los permisos de sudo de cada usuario en el sistema -

       https://imgur.com/SEtC1rsDemasiado tiempo para mostrar

/ home: el directorio donde se encuentran todas sus descargas, documentos, etc. - El equivalente en Windows es C: \ Users \ <user>

    

/ root: el directorio de inicio del usuario raíz: el equivalente en Windows es C: \ Users \ Administrator

       No se muestra por spoiler;)

/ usr: donde está instalado todo su software 

        

https://imgur.com/e0l4Ybh/ bin y / sbin - Usado para archivos críticos del sistema - NO ELIMINAR

    Demasiado grande para mostrar, pero contiene todos los programas básicos necesarios para que Linux funcione

/ var: el directorio misceláneo de Linux, una gran cantidad de procesos almacenan datos en / var

    

$ PATH: almacena todos los binarios que puede ejecutar, igual que $ PATH en Windows

    $ PATH es una variable de entorno que contiene todos los binarios que puede ejecutar. 

    

    Vale la pena señalar que las rutas en $ PATH (¡ja!) Están separadas por dos puntos. Cada archivo ejecutable que se encuentra en cualquiera de 
    esas rutas puede ejecutar simplemente escribiendo el nombre del ejecutable en lugar de la ruta completa.

    

     Nota: que es solo un comando que le muestra dónde está un ejecutable en cualquiera de los directorios PATH.
